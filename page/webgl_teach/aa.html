第一章——第一节

本节你能学到：

    （1）不同浏览器对于WebGL的支持情况
    （2）搭建合适的开发环境并测试一些实例

        WebGL目前作为Html5的准3D标准，越来越多的浏览器开始支持WebGL。使用WebGL，你可以直接利用显卡的处理资源，创造高性能2D和3D计算机图形。但是你要提前知道，直接用JavaScript进行WebGL编程来创建3D场景和动画是一个复杂且容易出错流程，学习难度会远大于通过使用Three.js来进行WebGL的编程。你可以做如下选择：

1.直接学习WebGL原生的编程过程

2.跳过WebGL的教学，先学习Three.js

        如果你选择直接学习WebGL的原生编程过程，你会了解到WebGL的本质。如果你想要先学Three.js，你会更加轻松且高效的运用WebGL做出超酷的项目与作品。无论选择哪种方式，你都可以随时去看另一部分教程。无论是使用WebGL原生的方式，还是使用Three.js，两者可以混合在项目中使用，因此当你学完某一方式的一部分内容后，可以去了解另一种方式，体会不同之处，各自的优缺点。

        在我们开始之前，让我们快速看最重要的浏览器那部分以及他们对WebGL的支持：

浏览器支持情况

Mozilla Firefox 4.0版本及以上.
Google Chrome 9.0版本及以上.
Safari 5.1版本及以上(仅Mac上支持。windows版的Safari不支持WebGL)请确保您允许WebGL在Safari浏览器中运行。您可以设置浏览器选项，在菜单Preferences|Advanced，检查在菜单栏中显示的Show develop menu选项。之后选择Develop | Enable WebGL。
Opera 12.0版本及以后的版本支持。你仍然需要通过打开文件：opera:config并且设置WebGL的Enable Hardware Acceleration为1来允许。之后重启浏览器。
Internet Explorer Internet Explorer已经是主流浏览器中很长时间不支持 WebGL。从IE11开始，微软加入了对WebGL的支持。

        基本上，WebGL可以运行在任何现代浏览器，除了IE以外的大多数版本。所以， 如果你想使用IE浏览器的旧版本，你有两个选择：你可以通过使用谷歌浏览器内嵌框架得到的WebGL，从这里下载支持。您还可以使用iewebgl插件， 你可以从 这里下载，把它安装在IE浏览器内并启用WebGL。

        在建立第一个场景的过程中，你将了解Three.js的基础知识，并创建你的第一个动画。在你对这个例子开始工作前，我们将首先看一下你所需要哪些开发工具。

        Three.js是一个JavaScript库，因此所有你所需要创建Three.js的WebGL应用程序的工具是一个文本编辑器和支持的浏览器来呈现结果。我想推荐4个不错的JavaScript编辑器：

● WebStorm：这个来自JetBrains导向的编辑器对于编辑JavaScript有很好的支持。它支持代码补全，自动部署，和JavaScript直接从编辑器调试。除此之外，WebStorm具有优良的Git支持（和其他版本控制系统）。你可以从这里下载试用版。

● Notepad++：这是一个通用的编辑器，支持多种编程语言。它可以很容易地铺陈和格式化JavaScript。你可以从这里下载。【Notepad++ 只能运行在Windows系统上】

● Sublime Text Editor：这是一个强大的且小巧的编辑器，它很好的支持编辑JavaScript。除了这一点，它提供了许多非常有用的选择和编辑选项，一旦你习惯了，它就提供一个真正好的JavaScript编辑环境。Sublime Text Editor也可以用于免费的测试。可以从这里下载。

● HBuilder：这是个专门为开发HTML5而设计的基于Eclipse内核的IDE，拥有强大的语法提示和代码高亮，（支持js，css，html），提供良好的测试工具。它能极大的提高你的开发效率。你可以从这里下载。【HBuilder 只能运行在Windows系统上】

        即使你不使用这4个编辑器，你仍然有很多开源和商业的编辑器可以选择。你可以用它来编辑JavaScript和创造你的Three.js项目。

        我曾提到，最现代的Web浏览器支持WebGL，可用于运行Three.js例子。我通常运行我在Chrome的代码。其原因在于:大多数情况下，Chrome浏览器对WebGL的支持和性能是最好的，它有一个真正伟大的JavaScript调试器。有了这个调试器，你可以快速查明问题，例如，通过使用断点和控制台输出。在教程中，我会给你调试程序的使用和其他的调试技巧和窍门。

        如果你的开发环境已经装好，那么你接下来要做的就是下载一些实例去测试运行。在Github上，Three.js项目为你提供了全部的开源代码，实例，API文档。你可以下载这个项目，解压后在examples文件夹内，可以点击里面的html文件进行测试。然而，你会发现有相当一部分实例你都无法正常运行，因为当实例中包含媒体文件，模型文件时，js通常需要通过http协议才能访问，而不能读取本地磁盘上的文件。因此我们需要搭建服务器。而搭建服务器则是极其容易的事情。下面将介绍几种常用服务器的搭建方法：

● 可以运行在大多数Unix/Mac系统上的基于Python的方法

大多数Unix/Linux/Mac系统已经内置了Python。在这些系统上，你可以很容易的通过以下指令开启服务器：
> python -m SimpleHTTPServer
Serving HTTP on 0.0.0.0 port 8000 ...
然后在你项目所在目录下执行。

● 如果你安装了Node.js，你可以使用基于NPM的方法

如果你已经用Node.js进行开发工作，你就已经安装了NPM。你有两种简单的选择来使用NPM快速建立本地服务器来测试。

第一种使用HTTP服务器模块：
> npm install -g http-server
> http-server
Starting up http-server, serving ./ on port: 8080
Hit CTRL-C to stop the server

你也可以使用Simple HTTP Server选项：
> npm install -g simple-http-server
> nserver
simple-http-server Now Serving: /Users/jos/git/Physijs at http://localhost:8000/

但是第二种方法有个缺点，那就是它不会自动显示目录列表，而第一种方式可以。

● 适用于Mac/ Windows便携版的Mongoose

Mongoose是个简单、便携的网络服务器。首先从这里为你特定平台下载二进制文件。在Windows平台上，把下载的文件复制到包含实例的目录下，双击它并启动浏览器，就能显示出它一开始所在目录中的内容。

对于其他操作系统，你必须把可执行文件复制到目标目录，你除了可以双击可执行文件，也可以用命令行启动它。在这两种情况下，一个本地服务器会在8080端口开启。

● 使用HBuilder内置的Apache服务器

如果你觉得安装并启动服务器麻烦，你只需要使用HBuilder作为你的开发工具就可以了。每当你使用HBuilder来开发，IDE内置的Apache服务器就会自动启动。你不需要再管服务器的任何事情。

        本节的内容到此结束。你现在拥有了可以开发Three.js的IDE，作为运行条件的服务器，支持WebGL的浏览器。你可以将实例代码导入进你的开发工具内，并运行实例。在下一节内，将正式进入代码部分，你将学习如何创建场景，添加各种几何物体，因此请确保你的IDE，服务器能正常工作，你的浏览器是支持WebGL的。对于浏览器的支持你现在就能判断，这个网站的每个页面的坐上方，都有各种颜色的动态3D新星（Nova），这些新星就是由WebGL所创建出来的，如果你能看到新星，那么你的浏览器完全没问题。
